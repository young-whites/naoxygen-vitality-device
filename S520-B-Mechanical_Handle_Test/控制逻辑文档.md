# S520-B 机械手柄控制逻辑文档

## 1. 系统概述

本项目基于 STM8S 微控制器，通过串口（UART1, 9600bps）接收上位机指令，控制直流电机的前进、后退、停止，并通过限位开关实现行程保护。

## 2. 硬件引脚定义

| 功能 | 引脚 | 说明 |
|------|------|------|
| ENA（电机使能） | PC6 | 高电平使能电机驱动器 |
| IN1（方向控制1） | PC4 | TIM1_CH2 PWM 输出 |
| IN2（方向控制2） | PC7 | TIM1_CH4 PWM 输出 |
| PC1（后端限位开关） | PD2 | 高电平触发 |
| PC2（前端限位开关） | PB4 | 高电平触发 |

## 3. 电机驱动原理

采用 ENA + IN1/IN2 方向控制方案：

| ENA | IN1 | IN2 | 动作 |
|-----|-----|-----|------|
| LOW | X | X | 停止（驱动器禁用） |
| HIGH | PWM | LOW | 前进 |
| HIGH | LOW | PWM | 后退 |
| HIGH | LOW | LOW | 制动 |

### 3.1 PWM 配置

- TIM1 时钟：16MHz，预分频 100 → 计数频率 158.4kHz
- PWM 模式：TIM1_OCMODE_TOGGLE（START）/ TIM1_OCMODE_PWM2（STOP）
- `speed_mode[0~3]`：前进速度档位（值越小速度越快）
- `speed_mode[4] = 0`：停止占位（不用于 PWM）
- `speed_mode[5] = 300`：后退固定速度

### 3.2 速度档位

| 档位 | 索引 | 速度值（示例） | 说明 |
|------|------|---------------|------|
| 档位 0 | `speed_mode[0]` | -0.1094×25212 + 5740 | 最慢 |
| 档位 1 | `speed_mode[1]` | -0.1094×25212 + 6200 | 慢 |
| 档位 2 | `speed_mode[2]` | -0.1094×25212 + 5850 | 默认 |
| 档位 3 | `speed_mode[3]` | -0.1094×25212 + 5350 | 最快 |
| 后退 | `speed_mode[5]` | 300（固定） | 后退专用 |

## 4. 系统启动流程

```
上电
 │
 ├─ disableInterrupts()
 ├─ CLK_HSIPrescalerConfig(HSIDIV1)    // 16MHz 时钟
 ├─ GPIO_Config()                       // 初始化所有 GPIO
 ├─ USARTAPP_Config()                   // 初始化 UART1 (9600, 8N1)
 ├─ TIM2_Config()                       // 初始化 TIM2 (1ms 中断)
 ├─ enableInterrupts()
 │
 ├─ motor 初始化 (direction=0, limit_rear=0, limit_front=0)
 ├─ Delay_ms(100)
 ├─ check_limit()                       // 【关键】上电立即检测限位状态
 ├─ Write_Option_Byte()
 ├─ EEPROM 初始化
 ├─ 计算 speed_mode[0~3]
 ├─ senddata(REVISION_CMD, 75)         // 上报版本号
 ├─ senddata(HANDLE_CHECK, 0)          // 上报手柄在线
 │
 └─ 进入主循环：while(1) { check_limit(); }
```

### 4.1 上电限位检测

上电后立即调用 `check_limit()`，如果电机初始位置已压在限位开关上：
- 对应限位标志立即置位
- 后续该方向的运动指令被拦截
- 仅允许反方向运动

## 5. 限位保护逻辑

### 5.1 限位状态机

```
                    ┌─────────────────────────────────┐
                    │         正常状态                  │
                    │   limit_front=0, limit_rear=0    │
                    │   前进/后退均可执行               │
                    └──────────┬──────────┬────────────┘
                               │          │
                    前端限位触发│          │后端限位触发
                               ▼          ▼
                ┌──────────────────┐  ┌──────────────────┐
                │  前端限位锁定     │  │  后端限位锁定     │
                │  limit_front=1   │  │  limit_rear=1    │
                │  前进被禁止       │  │  后退被禁止       │
                │  仅允许后退       │  │  仅允许前进       │
                └────────┬─────────┘  └────────┬─────────┘
                         │                     │
              电机后退离开 │           电机前进离开 │
              开关释放     │           开关释放     │
                         ▼                     ▼
                    ┌─────────────────────────────────┐
                    │         恢复正常状态              │
                    │   limit_front=0, limit_rear=0    │
                    └─────────────────────────────────┘
```

### 5.2 `check_limit()` 执行逻辑

在主循环中持续调用（每次 TIM2 中断后），检测 PC1/PC2 限位开关：

| 条件 | 动作 |
|------|------|
| PC1=HIGH 且 `limit_rear=0` | 设置 `limit_rear=1`，调用 `motor_stop()` |
| PC1=LOW 且 `direction==2`（前进） | 清除 `limit_rear=0`（电机已离开后端限位） |
| PC2=HIGH 且 `limit_front=0` | 设置 `limit_front=1`，调用 `motor_stop()` |
| PC2=LOW 且 `direction==1`（后退） | 清除 `limit_front=0`（电机已离开前端限位） |

**关键规则**：限位标志只在电机**反方向运动且开关释放**时清除，`motor_stop()` 不清除限位标志。

### 5.3 限位对运动指令的拦截

```c
motor_forward(mode):
    if (limit_front == 1) return;  // 前端限位激活时，前进指令被忽略

motor_backward():
    if (limit_rear == 1) return;   // 后端限位激活时，后退指令被忽略
```

## 6. 电机控制函数

### 6.1 `motor_stop()` — 停止

```c
void motor_stop(void)
{
    TIM1_DeInit();                    // 复位 TIM1
    TIM1_Cmd(DISABLE);                // 关闭 TIM1
    TIM1_CtrlPWMOutputs(DISABLE);    // 禁用 PWM 输出
    CLK_PeripheralClockConfig(CLK_PERIPHERAL_TIMER1, DISABLE);  // 关闭 TIM1 时钟

    GPIO_WriteLow(ENA);               // ENA = LOW（驱动器禁用）
    GPIO_Init(IN1, OUT_PP_LOW);       // IN1 = LOW
    GPIO_Init(IN2, OUT_PP_LOW);       // IN2 = LOW

    motor.direction = 0;
    // 注意：不清除 limit_front / limit_rear
}
```

**设计要点**：直接关闭 TIM1 而非调用 `pwm_init(STOP, 0, 0)`，避免 `speed_mode[4]=0` 导致 TIM1 period=0 引发 ~158kHz 中断风暴，阻塞 TIM2 串口解析。

### 6.2 `motor_forward(mode)` — 前进

```c
void motor_forward(u8 mode)
{
    if (motor.limit_front) return;    // 前端限位拦截

    motor.direction = 2;
    TIM1_DeInit();                    // 安全关闭 TIM1
    TIM1_Cmd(DISABLE);
    TIM1_CtrlPWMOutputs(DISABLE);

    GPIO_WriteLow(ENA);               // ENA LOW → HIGH（脉冲启动）
    GPIO_WriteHigh(ENA);
    pwm_init(START, speed_mode[mode], 2);  // 启动 PWM, direction=2
}
```

### 6.3 `motor_backward()` — 后退

```c
void motor_backward(void)
{
    if (motor.limit_rear) return;     // 后端限位拦截

    motor.direction = 1;
    TIM1_DeInit();                    // 安全关闭 TIM1
    TIM1_Cmd(DISABLE);
    TIM1_CtrlPWMOutputs(DISABLE);

    GPIO_WriteLow(ENA);               // ENA LOW → HIGH（脉冲启动）
    GPIO_WriteHigh(ENA);
    pwm_init(START, speed_mode[5], 1);  // 启动 PWM, direction=1, 固定速度
}
```

## 7. 串口指令处理

### 7.1 指令路由

```
串口数据 → UART1_RX_IRQHandler（入队）
        → TIM2_IRQHandler (1ms) → CommunicationWithMainBoard()
        → ReceiveMessageDecode() → DecodeMessageHandle()
        → CMD_MessageHandle() / SENDDATA_STATUS_MessageHandle()
```

### 7.2 下行指令处理

#### 7.2.1 电机运动模式 `0x17`（MOTOR_MODE_CMD）

| DataField[2] | 动作 |
|--------------|------|
| 0 或 5 | `motor_stop()` |
| 1~4 | `motor_forward(motor_mode)` |

#### 7.2.2 电机控制 `0x2D`（MOVE_CMD）

| DataField[2] | 动作 | 说明 |
|--------------|------|------|
| 0 | `motor_stop()` | 停止 |
| 1 | `motor_backward()` | 后退 |
| 2 | `motor_forward(motor_mode)` | 前进（使用当前速度档位） |
| 3 | `motor_backward()` + `senddata(0x2F, 1)` | 复位（后退 + 通知上位机） |
| 4 | `motor_backward()` + `Delay_ms(600)` + `motor_stop()` | 后退 0.6s 后停止 |

#### 7.2.3 速度切换 `0x15`（LITTLE_CMD）

| DataField[2] | motor_mode | 说明 |
|--------------|------------|------|
| 5 | 0 | 最慢 |
| 11~13 | 1 | 慢 |
| 20 | 2 | 默认 |
| 27~29 | 3 | 最快 |

#### 7.2.4 应答确认 `0x1C`（ACK_CMD）

回复 `HANDLE_CHECK(0xBB)` + 数据 `0x00`。

### 7.3 上行数据

| 功能码 | 说明 |
|--------|------|
| `0x19` | 版本号（当前值 75） |
| `0xBB` | 手柄在线应答 |
| `0x2F` | 退回标志（0=已退回，1=退回中） |

## 8. 中断架构

| 中断源 | 频率 | 功能 |
|--------|------|------|
| TIM2 更新中断 | 1ms | 调用 `CommunicationWithMainBoard()` 解析串口数据 |
| UART1 RX 中断 | 每字节 | 接收串口数据入队 |
| TIM1 更新中断 | PWM 周期 | PWM 操作（正常运行时触发） |

## 9. 关键设计决策

### 9.1 为什么 `motor_stop()` 不调用 `pwm_init(STOP, 0, 0)`？

`speed_mode[4] = 0`，调用 `pwm_init(STOP, 0, 0)` 会将 TIM1 period 设为 0，导致：
- TIM1 计数器每个时钟周期溢出
- TIM1 更新中断以 ~158kHz 频率连续触发
- TIM2 中断（串口解析）被完全阻塞
- 系统无法接收任何串口指令

解决方案：直接 `TIM1_DeInit()` + `TIM1_Cmd(DISABLE)` 关闭 TIM1。

### 9.2 为什么 `motor_stop()` 不清除限位标志？

限位标志用于锁定运动方向。如果 `motor_stop()` 清除限位标志：
- 限位触发后停止 → 标志清除 → 前进指令可再次执行 → 电机再次撞限位
- 无法实现"限位后只允许反方向运动"的保护逻辑

正确行为：限位标志在电机反方向运动且开关释放时由 `check_limit()` 清除。

### 9.3 为什么上电时调用 `check_limit()`？

上电时 `limit_front = 0`，`limit_rear = 0`。如果电机初始位置已压在限位开关上：
- 不调用 `check_limit()` → 限位标志为 0 → 前进/后退指令均可执行 → 电机可能撞限位
- 调用 `check_limit()` → 限位标志立即置位 → 对应方向被锁定

## 10. 状态变量

| 变量 | 类型 | 说明 |
|------|------|------|
| `motor.direction` | `volatile u8` | 0=停止, 1=后退, 2=前进 |
| `motor.limit_front` | `volatile u8` | 前端限位标志（1=锁定，0=正常） |
| `motor.limit_rear` | `volatile u8` | 后端限位标志（1=锁定，0=正常） |
| `motor_mode` | `u8` | 当前速度档位（0~3，默认 2） |
| `speed_mode[6]` | `u32` | 速度数组（索引 0~3 前进档位，4=0 停止占位，5=300 后退） |
