# S520-B Mechanical Handle Serial Communication Protocol

## 1. Basic Parameters

| Parameter | Value |
|-----------|-------|
| Serial Port | UART1 |
| Baud Rate | 9600 bps |
| Data Format | 8N1 (8 data bits, no parity, 1 stop bit) |
| Device ID | 0x00 0x01 |

## 2. Frame Format

### 2.1 Receive Frame (Host тЖТ Handle)

| Offset | Field | Length | Description |
|--------|-------|--------|-------------|
| 0-1 | FrameHeader | 2B | Fixed header `0x55 0xAA` |
| 2 | DataLength | 1B | Data length (includes DeviceID + CmdType + CmdStatus + DataField; excludes header, length, CRC) |
| 3-4 | DeviceID | 2B | Device ID, high byte `0x00`, low byte `0x01` |
| 5 | CmdType | 1B | Command type |
| 6 | CmdStatus | 1B | Command status |
| 7~N | DataField | Variable | Data payload |
| N+1~N+2 | CRC16 | 2B | CRC16 checksum (starting from DataLength field) |

### 2.2 Transmit Frame (Handle тЖТ Host)

Same format as above, DeviceID fixed to `0x00 0x01`.

## 3. Command Type (CmdType)

| CmdType | Macro | Description |
|---------|-------|-------------|
| `0x31` | SENDCMD | Send command (Host тЖТ Handle) |
| `0x32` | SENDDATA_STATUS | Send data/status (Bidirectional) |

## 4. Command Status (CmdStatus)

| CmdStatus | Macro | Description |
|-----------|-------|-------------|
| `0x02` | NORMAL_DATA_SEND | Normal data send |

## 5. Function Codes (DataField[0] = CmdType, DataField[1] = Function Code)

### 5.1 Downstream Commands (Host тЖТ Handle)

#### 5.1.1 Motor Motion Mode `0x17` тАФ MOTOR_MODE_CMD

Control motor step-forward or stop.

| DataField[2] | Action | Description |
|--------------|--------|-------------|
| `0` or `5` | Stop | Calls `motor_stop()` |
| `1` | Step forward | `k_flag=0`, auto-reverse at front end |
| `2` | Step forward | Same as above |
| `3` | Step forward | Same as above |
| `4` | Step forward | Same as above |

> Prerequisite: `stop_flog != 2` (not at front end)

#### 5.1.2 Motor Control `0x2D` тАФ MOVE_CMD

Manual control: forward, backward, stop, reset.

| DataField[2] | Action | Description |
|--------------|--------|-------------|
| `0` | Stop | Calls `motor_stop()` |
| `1` | Backward | Calls `motor_bank()`, prerequisite: `stop_flog != 1` |
| `2` | Manual forward | `k_flag=1`, calls `motor_forword(5)`, prerequisite: `stop_flog != 2` |
| `3` | Reset | Calls `motor_bank()` + `SendRuturnFlag(1)` |
| `4` | Backward 0.6s then stop | Calls `motor_bank()`, delays ~0.6s, then `motor_stop()`, position +252 |

#### 5.1.3 Motor Calibration `0x3D` тАФ CHECK_MOTOR_CMD

Calibrate motor travel. `k_flag=2`, uses speed_mode[5] (slowest speed).

| Condition | Action |
|-----------|--------|
| `stop_flog != 2` | Forward calibration `motor_forword(5)` |
| `stop_flog == 2` | Already at front, reverse `motor_bank()` |

#### 5.1.4 Acknowledge `0x1C` тАФ ACK_CMD

| Action | Description |
|--------|-------------|
| `ack_flag=1` | Mark as acknowledged |
| Reply `HANDLE_CHECK(0xBB)` | Handle detection response |

#### 5.1.5 Speed Trim / Mode Switch `0x15` тАФ LITTLE_CMD

Adjust motor speed level.

| DataField[2] | Speed Level | speed_mode Index | Approx. Speed |
|--------------|-------------|------------------|---------------|
| `5` | Level 0 | `speed_mode[0]` | ~11 |
| `11~13` | Level 1 | `speed_mode[1]` | ~10 |
| `20` | Level 2 | `speed_mode[2]` | ~9 |
| `27~29` | Level 3 | `speed_mode[3]` | ~8 |

#### 5.1.6 Motor Progress Save `0x3E` тАФ MOTOR_STEP_SAVE

Save current motor position `tim1_count_cnt1` to EEPROM.

#### 5.1.7 Speed Calibration `0xED` тАФ Speed_Adjust_Cmd

| DataField[2] | DataField[3] | Description |
|--------------|--------------|-------------|
| `adjust_mode` | `adjust_Value` | Written to EEPROM; mode=1 negates value |

After calibration, recalculates `speed_mode[0~3]`:
```
speed_mode[i] = -0.1094 * max_motor_count + adjust_Value + base_value
```

#### 5.1.8 Return Flag Set `0xF5`

| DataField[2] | Description |
|--------------|-------------|
| Value | Directly assigned to `IfRuturning` |

### 5.2 Upstream Data (Handle тЖТ Host)

#### 5.2.1 Motor Position `0x2E` тАФ MOTOR_STEP_CMD

| DataField[0] | DataField[1] | Description |
|--------------|--------------|-------------|
| `0x2E` | `0~100` | Motor position percentage (0=front, 100=rear) |

Formula: `number = 100 - tim1_count_cnt1 / motor_100`

#### 5.2.2 Return Flag `0x2F` тАФ RUTURN_FLAG_CMD

| DataField[1] | Description |
|--------------|-------------|
| `0` | Motor returned to rear end, commands accepted |
| `1` | Motor returning, commands rejected |

#### 5.2.3 Handle Detection Response `0xBB` тАФ HANDLE_CHECK

| DataField[1] | Description |
|--------------|-------------|
| `0` | Handle online acknowledgment |

#### 5.2.4 Version Number `0x19` тАФ REVISION_CMD

| DataField[1] | Description |
|--------------|-------------|
| `75` | Current firmware version |

#### 5.2.5 Query Motor Position `0x2E`

When CmdType=`0x32`(SENDDATA_STATUS) and DataField=`0x2E`, triggers motor position report.

## 6. Power-On Initialization Flow

```
1. Power-on тЖТ GPIO_Config() тЖТ ENA HIGH, IN1/IN2 push-pull output
2. Read EEPROM to check first boot
   - First boot: write default values
   - Not first boot: restore adjust_mode, adjust_Value, motor position
3. Calculate speed_mode[0~3] based on adjust_mode
4. Report motor position SendDjData()
5. Report version senddata(REVISION_CMD, 75)
6. Report handle detection senddata(HANDLE_CHECK, 0)
7. Enter main loop check_stop()
```

## 7. Motor Stop Logic (ENA Always-High Scheme)

```
motor_stop():
  1. pwm_init(STOP) тЖТ TIM1 output disabled
  2. motor_hiz() тЖТ IN1/IN2 switched to floating input (Hi-Z)
  3. ENA remains HIGH
```

## 8. Limit Switches

| Switch | Pin | Trigger | Action |
|--------|-----|---------|--------|
| PC1 (Rear) | PD2 | High | Stop + rear calibration |
| PC2 (Front) | PB4 | High | Stop + front calibration (auto-reverse in step mode) |
