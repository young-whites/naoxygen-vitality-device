# S520-B 机械手柄串口通信协议文档

## 1. 基本参数

| 参数 | 值 |
|------|-----|
| 串口 | UART1 |
| 波特率 | 9600 bps |
| 数据格式 | 8N1（8数据位，无校验，1停止位） |
| 设备 ID | 0x00 0x01 |

## 2. 帧格式

### 2.1 接收帧（主机 → 手柄）

| 偏移 | 字段 | 长度 | 说明 |
|------|------|------|------|
| 0-1 | FrameHeader | 2B | 固定帧头 `0x55 0xAA` |
| 2 | DataLength | 1B | 数据长度（含 DeviceID + CmdType + CmdStatus + DataField，不含帧头、长度、CRC） |
| 3-4 | DeviceID | 2B | 设备 ID，高字节 `0x00`，低字节 `0x01` |
| 5 | CmdType | 1B | 命令类型 |
| 6 | CmdStatus | 1B | 命令状态 |
| 7~N | DataField | 变长 | 数据域 |
| N+1~N+2 | CRC16 | 2B | CRC16 校验（从 DataLength 字段起算） |

### 2.2 发送帧（手柄 → 主机）

格式同上，DeviceID 固定为 `0x00 0x01`。

## 3. 命令类型（CmdType）

| CmdType | 宏定义 | 说明 |
|---------|--------|------|
| `0x31` | SENDCMD | 发送命令（主机 → 手柄） |
| `0x32` | SENDDATA_STATUS | 发送数据/状态（双向） |

## 4. 命令状态（CmdStatus）

| CmdStatus | 宏定义 | 说明 |
|-----------|--------|------|
| `0x02` | NORMAL_DATA_SEND | 普通数据发送 |

## 5. 功能码（DataField[0] = CmdType, DataField[1] = 功能码）

### 5.1 下行命令（主机 → 手柄）

#### 5.1.1 电机运动模式 `0x17` — MOTOR_MODE_CMD

控制电机按步骤前进或停止。

| DataField[2] | 动作 | 说明 |
|--------------|------|------|
| `0` 或 `5` | 停止 | 调用 `motor_stop()` |
| `1` | 步骤前进 | `k_flag=0`，到达前端自动退回 |
| `2` | 步骤前进 | 同上 |
| `3` | 步骤前进 | 同上 |
| `4` | 步骤前进 | 同上 |

> 前提条件：`stop_flog != 2`（不在前端时才执行前进）

#### 5.1.2 电机控制 `0x2D` — MOVE_CMD

手动控制电机前进、后退、停止、复位。

| DataField[2] | 动作 | 说明 |
|--------------|------|------|
| `0` | 停止 | 调用 `motor_stop()` |
| `1` | 后退 | `k_flag` 不变，调用 `motor_bank()`，前提 `stop_flog != 1` |
| `2` | 手动前进 | `k_flag=1`，调用 `motor_forword(5)`，前提 `stop_flog != 2` |
| `3` | 复位 | 调用 `motor_bank()` + 通知主板电机正在退回 `SendRuturnFlag(1)` |
| `4` | 退回0.6s后停 | 调用 `motor_bank()`，延时约 0.6s 后 `motor_stop()`，位置计数 +252 |

#### 5.1.3 电机校验 `0x3D` — CHECK_MOTOR_CMD

校准电机行程，`k_flag=2`，使用 speed_mode[5]（最慢速）前进到前端后自动退回。

| 条件 | 动作 |
|------|------|
| `stop_flog != 2` | 前进校验 `motor_forword(5)` |
| `stop_flog == 2` | 已在前端，直接后退 `motor_bank()` |

#### 5.1.4 应答确认 `0x1C` — ACK_CMD

| 动作 | 说明 |
|------|------|
| `ack_flag=1` | 标记已应答 |
| 回复 `HANDLE_CHECK(0xBB)` | 手柄检测应答 |

#### 5.1.5 速度微调/模式切换 `0x15` — LITTLE_CMD

调节电机运行速度档位。

| DataField[2] | 速度档位 | speed_mode 索引 | 对应速度 |
|--------------|----------|-----------------|----------|
| `5` | 档位 0 | `speed_mode[0]` | 约 11 级 |
| `11~13` | 档位 1 | `speed_mode[1]` | 约 10 级 |
| `20` | 档位 2 | `speed_mode[2]` | 约 9 级 |
| `27~29` | 档位 3 | `speed_mode[3]` | 约 8 级 |

#### 5.1.6 电机进度保存 `0x3E` — MOTOR_STEP_SAVE

将当前电机位置 `tim1_count_cnt1` 保存到 EEPROM。

#### 5.1.7 速度校准 `0xED` — Speed_Adjust_Cmd

| DataField[2] | DataField[3] | 说明 |
|--------------|--------------|------|
| `adjust_mode` | `adjust_Value` | 写入 EEPROM，mode=1 时取负值补偿 |

校准后重新计算 `speed_mode[0~3]`：
```
speed_mode[i] = -0.1094 * max_motor_count + adjust_Value + 基准值
```

#### 5.1.8 回退标记设置 `0xF5`

| DataField[2] | 说明 |
|--------------|------|
| 值 | 直接赋值给 `IfRuturning` |

### 5.2 上行数据（手柄 → 主机）

#### 5.2.1 电机位置 `0x2E` — MOTOR_STEP_CMD

| DataField[0] | DataField[1] | 说明 |
|--------------|--------------|------|
| `0x2E` | `0~100` | 电机当前位置百分比（0=前端，100=后端） |

计算公式：`number = 100 - tim1_count_cnt1 / motor_100`

#### 5.2.2 退回标志 `0x2F` — RUTURN_FLAG_CMD

| DataField[1] | 说明 |
|--------------|------|
| `0` | 电机已退回后端，可以接受 APP 指令 |
| `1` | 电机正在退回中，APP 指令不生效 |

#### 5.2.3 手柄检测应答 `0xBB` — HANDLE_CHECK

| DataField[1] | 说明 |
|--------------|------|
| `0` | 手柄在线应答 |

#### 5.2.4 版本号 `0x19` — REVISION_CMD

| DataField[1] | 说明 |
|--------------|------|
| `75` | 当前固件版本号 |

#### 5.2.5 获取电机位置 `0x2E`

当 CmdType=`0x32`(SENDDATA_STATUS)、DataField=`0x2E` 时，触发上报当前电机位置。

## 6. 上电初始化流程

```
1. 上电 → GPIO_Config() → ENA 拉高，IN1/IN2 推挽输出
2. 读取 EEPROM 判断是否首次启动
   - 首次：写入默认值
   - 非首次：恢复 adjust_mode、adjust_Value、电机位置
3. 根据 adjust_mode 计算 speed_mode[0~3]
4. 上报电机位置 SendDjData()
5. 上报版本号 senddata(REVISION_CMD, 75)
6. 上报手柄检测 senddata(HANDLE_CHECK, 0)
7. 进入主循环 check_stop()
```

## 7. 电机停止逻辑（ENA 常高方案）

```
motor_stop():
  1. pwm_init(STOP) → TIM1 输出关闭
  2. motor_hiz() → IN1/IN2 切换为浮空输入（高阻态）
  3. ENA 保持高电平不变
```

## 8. 限位开关

| 开关 | 引脚 | 触发 | 动作 |
|------|------|------|------|
| PC1（后端） | PD2 | 高电平 | 停止 + 后端校准 |
| PC2（前端） | PB4 | 高电平 | 停止 + 前端校准（步骤模式自动退回） |
