# S520-B 通信协议解析

## 1. 协议概述

S520-B 系统中存在两套通信协议：

1. **自定义帧协议**：主控板 ↔ 蓝牙APP、主控板 ↔ 机械手柄
2. **Ymodem 协议**：Bootloader ↔ 上位机（固件升级）

---

## 2. 自定义帧协议（0x55AA 协议）

### 2.1 帧格式

```
┌──────┬──────┬─────┬───────┬───────┬─────────┬──────────┬──────────┬──────────┬──────────┐
│ 帧头1 │ 帧头2 │ 长度 │ ID_H  │ ID_L  │ CmdType │ CmdStatus│ Data[N]  │ CRC16_H  │ CRC16_L  │
│ 0x55  │ 0xAA │ Len │ 0x00  │ 0x01  │  Type   │  Status  │  N字节   │  高字节   │  低字节   │
└──────┴──────┴─────┴───────┴───────┴─────────┴──────────┴──────────┴──────────┴──────────┘
```

### 2.2 帧字段定义

| 字段 | 偏移 | 长度 | 说明 |
|------|------|------|------|
| FrameHeader | 0 | 2B | 帧头固定为 0x55AA |
| DataLength | 2 | 1B | 数据长度（从 DeviceID 到 Data 末尾，含 CmdType/CmdStatus/Data） |
| DeviceID_H | 3 | 1B | 设备ID高字节，固定 0x00 |
| DeviceID_L | 4 | 1B | 设备ID低字节，固定 0x01 |
| CmdType | 5 | 1B | 命令类型 |
| CmdStatus | 6 | 1B | 命令状态/方向 |
| Data | 7 | NB | 数据域（长度 = DataLength - 4） |
| CRC16_H | 7+NB | 1B | CRC16 校验高字节 |
| CRC16_L | 8+NB | 1B | CRC16 校验低字节 |

**CRC 计算范围**：从 DataLength 字段开始到 Data 末尾（即偏移2到偏移6+NB）

### 2.3 命令类型（CmdType）

| 值 | 宏定义 | 说明 |
|----|--------|------|
| 0x31 | `FRAME_TYPE_ACT` | 动作命令（控制设备执行操作） |
| 0x32 | `FRAME_TYPE_SET` | 参数设置（写入配置参数） |
| 0x33 | `FRAME_TYPE_GET` | 参数获取（读取设备状态） |
| 0x66 | `FRAME_TYPE_POST` | 主动上报（设备主动推送数据） |

### 2.4 命令状态（CmdStatus）

| 值 | 宏定义 | 说明 |
|----|--------|------|
| 0x02 | `FRAME_STATE_ASK` | 上位机请求（APP/蓝牙发送） |
| 0x01 | `FRAME_STATE_ACK` | 下位机应答（设备回复） |
| 0x00 | `FRAME_STATE_ERR` | 校验出错 |

### 2.5 CRC16-Modbus 校验

**算法参数：**
- 初始值：0xFFFF
- 多项式：0xA001（反转表示，实际为 X¹⁶+X¹⁵+X²+1）
- 输入反转：是
- 输出反转：是

**C 语言实现：**
```c
uint16_t CrcCalc_Crc16Modbus(uint8_t *dat, uint8_t len)
{
    uint16_t CRC_index = 0xffff;
    uint16_t buffer;
    uint8_t i, j;
    for (i = 0; i < len; i++) {
        buffer = dat[i];
        CRC_index ^= buffer;
        for (j = 0; j < 8; j++) {
            if (CRC_index & 0x0001) {
                CRC_index >>= 1;
                CRC_index ^= 0xa001;
            } else {
                CRC_index >>= 1;
            }
        }
    }
    return CRC_index;
}
```

## 3. 主控 ↔ 蓝牙/APP 通信协议

### 3.1 物理层

| 参数 | 值 |
|------|-----|
| 串口 | USART2 |
| 波特率 | 9600bps |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | 无 |
| 流控 | 无 |
| DMA | TX: DMA1 Ch7, RX: DMA1 Ch6 |

### 3.2 协议帧

帧格式见第2节，设备ID为 0x0001。

### 3.3 功能码定义（CmdType=0x31, CmdStatus=0x02）

| 功能码 | 宏定义 | 方向 | 说明 | 数据域 |
|--------|--------|------|------|--------|
| 0x01 | `FRAME_APP_SET_WORK_SWITCH_CMD` | APP→主控 | 开关机指令 | [0x01=开, 0x00=关] |
| 0x02 | `FRAME_APP_SET_HAND_WORK_STATUS_CMD` | APP→主控 | 手柄睡眠状态 | — |
| 0x0D | `FRAME_APP_SET_FLOW_MODE_CMD` | APP→主控 | 配置流量模式 | — |
| 0x0E | `FRAME_APP_SET_PRESSURE_MODE_CMD` | APP→主控 | 配置压力模式 | — |
| 0x0F | `FRAME_APP_SET_TAKT_MODE_CMD` | APP→主控 | 配置节拍模式 | — |
| 0x15 | — | APP→主控 | 速度微调指令 | [档位值] |
| 0x29 | `FRAME_APP_SET_HAND_ADJUST_SPEED_CMD` | APP→主控 | 手柄调速 | — |
| 0x2C | `FRAME_APP_SET_MASTER_CYLINDER_CMD` | APP→主控 | 主气缸限压模式 | — |
| 0x2D | `FRAME_APP_SET_MACHINE_HAND_CMD` | APP→主控 | 机械推杆控制 | [方向: 0=停,1=退,2=进,3=复位,4=退停] |
| 0x2E | `FRAME_APP_SET_MOTOR_LOCATION_CMD` | APP→主控 | 设置电机位置 | — |
| 0x2F | `FRAME_APP_SET_HAND_RETURN_STATUS_CMD` | APP→主控 | 手柄退回状态 | — |
| 0xAA | `FRAME_APP_SET_SYS_UPDATE_CMD` | APP→主控 | 系统升级指令 | — |

### 3.4 接收解码流程

```
USART2 RX DMA → 环形缓冲区 → 主循环逐字节读取
  → 解码状态机 (8步)：
    Step 0: 匹配帧头0x55
    Step 1: 匹配帧头0xAA
    Step 2: 读取数据长度
    Step 3: 匹配设备ID高字节 0x00
    Step 4: 匹配设备ID低字节 0x01
    Step 5: 读取命令类型
    Step 6: 读取命令状态+数据域
    Step 7: CRC16校验 → 通过则调用 Deal_correct_data()
```

### 3.5 发送组帧流程

```c
void USART2_Send_Command_to_Principal(u8 Datalength, u8 Cmdtype, 
                                       u8 Cmdstatus, u8 *Datafile)
{
    send_data[0] = 0x55;           // 帧头1
    send_data[1] = 0xAA;           // 帧头2
    send_data[2] = 4 + Datalength; // 数据长度
    send_data[3] = 0x00;           // 设备ID高
    send_data[4] = 0x01;           // 设备ID低
    send_data[5] = Cmdtype;        // 命令类型
    send_data[6] = Cmdstatus;      // 命令状态
    // Datafile 填入 send_data[7..7+Datalength-1]
    // CRC16 填入 send_data[6+Datalength+1] 和 [+2]
    Uart_Send_Upat(&USART2_SendBuf, send_data, 7+Datalength+2);
}
```

## 4. 主控 ↔ 机械手柄通信协议

### 4.1 物理层

| 参数 | 值 |
|------|-----|
| 串口 | UART4 |
| 波特率 | 9600bps |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | 无 |
| 流控 | 无 |
| DMA | TX: DMA2 Ch5, RX: DMA2 Ch3 |

### 4.2 协议帧

帧格式与第2节完全相同，设备ID为 0x0001，使用相同的 CRC16-Modbus 校验。

### 4.3 主控 → 手柄命令（CmdType=0x31）

| 功能码 | 说明 | 数据域 |
|--------|------|--------|
| 0x17 | 工作模式（速度档位） | [档位: 0~5] |
| 0x15 | 速度微调 | [档位映射值] |
| 0x1C | ACK应答查询 | — |
| 0x2D | 电机控制 | [0=停, 1=退, 2=进, 3=复位, 4=退停] |
| 0x3D | 电机校验 | — |
| 0x3E | 保存电机位置 | — |
| 0xED | 速度补偿调节 | [模式, 补偿值] |
| 0xF5 | 回退状态设置 | [状态值] |

### 4.4 手柄 → 主控数据（CmdType=0x32, CmdStatus=0x02）

| 功能码 | 说明 | 数据域 |
|--------|------|--------|
| 0x19 | 版本号上报 | [类型, 版本号(75)] |
| 0x2E | 电机位置上报 | [类型, 位置百分比(0~100)] |
| 0xBB | 手柄检测应答 | [类型, 0] |
| 0x2F | 电机退回状态通知 | [类型, 0=已退回/1=正在退回] |

### 4.5 手柄电机位置上报

位置计算公式：
```c
number = 100 - (tim1_count_cnt1 / motor_100);
// tim1_count_cnt1: 当前步数计数器
// motor_100: 100%对应的步数 (252)
// number: 位置百分比 (0~100)
```

### 4.6 速度档位映射

| 档位值 | 含义 | PWM周期值 | 速度方向 |
|--------|------|-----------|---------|
| 0 | 停止 | 0 | — |
| 1 | 10速 | speed_mode[1] | 前进 |
| 2 | 9速 | speed_mode[2] | 前进 |
| 3 | 8速 | speed_mode[3] | 前进 |
| 4 | 11速 | speed_mode[0] | 前进 |
| 5 | 后退 | speed_mode[5] | 后退 |

速度档位计算公式：
```c
speed_mode[0] = (u32)(-0.1094 * max_motor_count + adjust_Value + 5740); // 11速
speed_mode[1] = (u32)(-0.1094 * max_motor_count + adjust_Value + 6200); // 10速
speed_mode[2] = (u32)(-0.1094 * max_motor_count + adjust_Value + 5850); // 9速
speed_mode[3] = (u32)(-0.1094 * max_motor_count + adjust_Value + 5350); // 8速
```

## 5. Bootloader Ymodem 协议

### 5.1 物理层

| 参数 | 值 |
|------|-----|
| 串口 | USART2 |
| 波特率 | 9600bps |
| 数据格式 | 8N1 |

### 5.2 协议控制字符

| 字符 | 值 | 说明 |
|------|-----|------|
| SOH | 0x01 | 128字节数据包开始 |
| STX | 0x02 | 1024字节数据包开始 |
| EOT | 0x04 | 传输结束 |
| ACK | 0x06 | 确认应答 |
| NAK | 0x15 | 否定应答 |
| CA | 0x18 | 中止传输（连续2个） |
| C | 0x43 | 请求CRC校验模式 |
| 'A' | 0x41 | 用户终止 |
| 'a' | 0x61 | 用户终止 |

### 5.3 数据包格式

#### 128字节包 (SOH)

```
┌──────┬──────┬───────┬──────────────┬──────────┬──────────┐
│ SOH  │ 序号  │ 序号补 │  数据(128B)   │ CRC16_H  │ CRC16_L  │
│ 0x01 │ Seq  │ ~Seq  │   Data       │          │          │
└──────┴──────┴───────┴──────────────┴──────────┴──────────┘
```

#### 1024字节包 (STX)

```
┌──────┬──────┬───────┬───────────────────┬──────────┬──────────┐
│ STX  │ 序号  │ 序号补 │    数据(1024B)     │ CRC16_H  │ CRC16_L  │
│ 0x02 │ Seq  │ ~Seq  │     Data          │          │          │
└──────┴──────┴───────┴───────────────────┴──────────┴──────────┘
```

### 5.4 CRC16-CCITT 校验

**算法参数：**
- 初始值：0
- 多项式：0x1021
- 不反转

**C 语言实现：**
```c
u16 UpdateCRC16(uint16_t crcIn, uint8_t byte) {
    uint32_t crc = crcIn;
    uint32_t in = byte | 0x100;
    do {
        crc <<= 1;
        in <<= 1;
        if (in & 0x100) ++crc;
        if (crc & 0x10000) crc ^= 0x1021;
    } while (!(in & 0x10000));
    return crc & 0xffff;
}
```

### 5.5 传输流程

```
发送端                          接收端(Bootloader)
  │                                │
  │──── SOH + 包0(文件信息) ──────>│
  │                                │ 验证文件名(MAJOR.bin/MINOR.bin)
  │                                │ 验证文件大小
  │<──────────── ACK ─────────────│
  │<──────────── C ───────────────│ (请求CRC模式)
  │                                │ 擦除Flash
  │──── STX + 包1(数据) ─────────>│
  │                                │ CRC校验
  │                                │ 写入Flash
  │<──────────── ACK ─────────────│
  │──── STX + 包2(数据) ─────────>│
  │<──────────── ACK ─────────────│
  │     ...                        │
  │──── EOT ──────────────────────>│
  │<──────────── ACK ─────────────│
  │                                │ 清除升级标志
  │                                │ 系统复位
```

### 5.6 错误处理

| 错误类型 | 返回值 | 处理 |
|----------|--------|------|
| 固件过大 | -1 | 发送 CA+CA 中止 |
| Flash烧写错误 | -2 | 发送 CA+CA 中止 |
| 用户终止 | -3 | 发送 CA+CA 中止 |
| CRC校验失败 | -4 | 发送 CA+CA 中止 |
| 序号错误 | -5 | 返回 NAK |
| 文件名错误 | -6 | 发送 CA+CA 中止 |
| 接收超时 | -1 | 发送 'C' 重试 |

## 6. 蓝牙AT指令

蓝牙模块上电后，主控板通过 USART2 发送AT指令设置蓝牙名称：

```
AT+NAMEX04\r\n
```

发送时机：上电后500ms（`btPowerOnCnt == 50`，扫描周期10ms）

---

## 7. 升级命令匹配

APP 发送升级指令时，使用特定字符串匹配机制：

```
升级命令: "FWUPDATE" (8字节)
匹配方式: 逐字节匹配，全部匹配后设置 Flag.update = 1
```

匹配完成后主控板：
1. 写入升级标志到 Flash (0x0801E000)
2. 喂狗
3. 跳转到系统 Bootloader (0x08000000)
